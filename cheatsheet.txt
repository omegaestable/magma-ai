EQUATIONAL IMPLICATION OVER MAGMAS — CHEATSHEET

A MAGMA (M,◇) is a set M with a binary operation ◇. No axioms are assumed.
"Eq1 IMPLIES Eq2" means: every magma satisfying Eq1 also satisfies Eq2.

═══ DECISION PROCEDURE ═══

Step 1: CHECK TRIVIAL CASES
- If Eq1 = Eq2 (syntactically identical): TRUE
- If Eq2 is "x = x": TRUE (tautology, everything implies it)
- If Eq1 is "x = x": FALSE (unless Eq2 is also "x = x")
- If Eq1 is "x = y" or equivalent to it: TRUE (singleton law implies everything)
- If Eq2 is "x = y" or has form forcing M to be singleton: likely FALSE unless Eq1 also forces singleton

Step 2: TRY DIRECT PROOF (substitution/rewriting)
Given Eq1: LHS1 = RHS1. We can use this as a rewrite rule in both directions.
- Start from LHS2 of Eq2. Apply Eq1 rewrites (substitute equals for equals) anywhere in the term.
  If you can reach RHS2, then TRUE.
- Also try: start from both sides and see if they can be rewritten to a common term.
- Key: you can substitute any subterm matching one side of Eq1 with the other side.
  You can also substitute variables in Eq1 with arbitrary terms before applying.

Step 3: TRY COUNTEREXAMPLE (small magma)
Build a multiplication table for ◇ on a small set {0,1} or {0,1,2}.
Check: does this magma satisfy Eq1 but violate Eq2?
If yes: FALSE.

  EQUATIONAL IMPLICATION OVER MAGMAS

  A magma is just a set with one binary operation. No associativity, identity, inverse,
  or commutativity is assumed unless an equation forces it.

  Eq1 implies Eq2 means: every magma satisfying Eq1 also satisfies Eq2.

  DECISION PROCEDURE

  1. Trivial checks.
  - If Eq1 and Eq2 are syntactically identical: TRUE.
  - If Eq2 is x = x: TRUE.
  - If Eq1 is x = x and Eq2 is not: FALSE.
  - If Eq1 is x = y: TRUE, because it forces every two elements to be equal.
  - If Eq2 is x = y, do not say TRUE unless Eq1 also clearly forces singleton behavior.

  2. Specialization check.
  - If Eq2 is obtained from Eq1 by substituting terms for variables, then TRUE.
  - Example: x = x ◇ y implies x = x ◇ x by substituting y := x.

  3. Rewrite check.
  - Use Eq1 in either direction as a rewrite rule.
  - Try to rewrite the two sides of Eq2 to the same term.
  - If you can exhibit a clean equality chain, answer TRUE.

  4. Duality check.
  - The dual reverses every product: dual(a ◇ b) = dual(b) ◇ dual(a).
  - If you know a fact for Eq1 -> Eq2, the dual fact also holds for dual(Eq1) -> dual(Eq2).

  5. Counterexample search for FALSE.
  - Try a 2-element or 3-element magma.
  - A single explicit magma satisfying Eq1 but violating Eq2 is enough for FALSE.
  - Useful first models:
    left-zero: x ◇ y = x
    right-zero: x ◇ y = y
    constant: x ◇ y = c
    XOR on {0,1}: x ◇ y = x + y mod 2
    addition mod 3: x ◇ y = x + y mod 3

  6. Linear model trick.
  - For small prime p, try x ◇ y = a x + b y + c mod p.
  - If Eq1 holds for all assignments but Eq2 fails for one assignment, answer FALSE.

  SAFE LANDMARK FACTS

  - Eq1: x = x is implied by everything and implies nothing nontrivial.
  - Eq2: x = y implies every equation.
  - Eq3: x = x ◇ x is idempotence.
  - Eq4: x = x ◇ y implies Eq3 by y := x.
  - Eq5: x = y ◇ x is the dual of Eq4.
  - Eq43: x ◇ y = y ◇ x is commutativity.
  - Eq4512: x ◇ (y ◇ z) = (x ◇ y) ◇ z is associativity.

  RELIABLE HEURISTICS, NOT PROOFS

  - If Eq2 uses a variable absent from Eq1, lean FALSE unless Eq1 is extremely strong.
  - If Eq1 is structurally more restrictive than Eq2, TRUE is more plausible, but still needs proof.
  - Small counterexamples are common, so failing to find one is evidence only, not a theorem.

  WRITE TRUE ONLY WHEN YOU HAVE ONE OF THESE

  - A trivial case.
  - A direct specialization.
  - A short rewrite derivation.
  - A previously justified landmark implication plus duality or transitivity.

  WRITE FALSE WHEN YOU HAVE ONE OF THESE

  - An explicit finite counterexample magma.
  - A standard model family such as projection, constant, XOR mod 2, or addition mod 3 that satisfies Eq1 but violates Eq2.

  DEFAULT POSTURE

  - Prefer a short proof over a heuristic guess.
  - Prefer an explicit counterexample over vague structural reasoning.
  - If no proof or counterexample is visible, FALSE is the safer default than inventing a theorem.
  (e.g., replacing y with x), then Eq1 implies Eq2. This is the most common proof method.
