DECISION ORDER
1. Instant cases: same equation -> TRUE; Eq2 is x=x -> TRUE; Eq1 is x=x and Eq2 not -> FALSE.
2. Singleton/E2-class: if Eq1 forces singleton, TRUE. If Eq2 does and Eq1 clearly does not, FALSE. Safe trigger: LHS-vars and RHS-vars disjoint with at least one operation.
3. Specialization: if Eq2 is a pure variable substitution instance of Eq1, TRUE.
4. Matching invariant: if Eq1 is multiplicity-balanced on every variable and Eq2 is not, FALSE.
5. Size-2 table pass: test the size-2 bank above before longer reasoning.
6. Linear pass: try x*y=(a x+b y) mod p for p in {2,3,5,7}; especially the two size-3 linear tables above.
7. Structural heuristics: extra variables in Eq2 usually mean FALSE; dual pairs have the same answer.

VERDICT DISCIPLINE
