  YES: "x = ((y * z) * w) * x"    RHS ends "* x"

  NO:  "x = (y * x) * (x * (z * z))"  RHS ends "z))" not "* x"
  NO:  "x = ((x * x) * (y * z)) * y"  RHS ends "* y" not "* x"
  If RHS ends "* x": RP holds E1 trivially (rightmost leaf = x).
  Also check E2: is rightmost leaf of E2's RHS = x? If YES → VERDICT: TRUE.

=== COUNTEREXAMPLE VERIFICATION PROTOCOL (MANDATORY) ===
BEFORE claiming any magma M as counterexample for "E1 implies E2":
1. CHECK E1 holds on ALL inputs of M. If E1 FAILS on M, M is INVALID — stop.
2. CHECK E2 fails on SOME input of M. If E2 holds on all, M is NOT a CE.
Only if both steps pass: M is a valid counterexample, VERDICT FALSE.
If no valid CE found after checking: lean toward TRUE, not FALSE.
CRITICAL WARNING: Most equations with 3+ variables (free variable z, w, etc.)
are very restrictive. Small magmas (LP, RP, XOR) satisfy very FEW of them.
Do NOT assume a simple magma satisfies a complex equation — ALWAYS verify.

=== CANONICAL MAGMAS (Cayley tables, read as T[row][col]) ===

LP (Left Projection): x*y = x for all x,y.
  Table (size 2): [[0,0],[1,1]]
  FAST EVAL RULE: on LP, A*B = A always. So any nested expr = its LEFTMOST
  LEAF variable in the parse tree (ignore all right-side arguments).
    x*(y*(x*z))     → leftmost leaf = x → evaluates to x
    ((y*y)*z)*x     → leftmost leaf = y → evaluates to y  ← NOT x!
    y*(y*(z*(w*x))) → leftmost leaf = y → evaluates to y  ← NOT x!
  QUICK TEST for "x = RHS": find leftmost leaf of RHS.
    leaf = x  → LP might satisfy E1 (verify E2 next)
    leaf ≠ x  → LP FAILS E1. Do NOT use LP as CE.

RP (Right Projection): x*y = y for all x,y.
  Table (size 2): [[0,1],[0,1]]
  FAST EVAL RULE: on RP, A*B = B always. So any nested expr = its RIGHTMOST
  LEAF variable in the parse tree (ignore all left-side arguments).
    ((y*y)*z)*x     → rightmost leaf = x → evaluates to x
    x*(y*(x*z))     → rightmost leaf = z → evaluates to z  ← NOT x!
    (y*x)*(x*(z*z)) → rightmost leaf = z → evaluates to z  ← NOT x!
  QUICK TEST for "x = RHS": find rightmost leaf of RHS.
    leaf = x  → RP might satisfy E1 (verify E2 next)
    leaf ≠ x  → RP FAILS E1. Do NOT use RP as CE.

XOR (Z/2Z addition): x*y = (x+y) mod 2.
  Table (size 2): [[0,1],[1,0]]
  Properties: commutative, associative, x*x = 0, x*0 = x.
  XOR satisfies very few equational laws with free variables.

Z3A (Z/3Z addition): x*y = (x+y) mod 3.
  Table (size 3): [[0,1,2],[1,2,0],[2,0,1]]
  Properties: commutative, associative, x*x*x = 0, order 3.
  Z3A satisfies very few equational laws with 3+ variables.

C0 (Constant zero): x*y = 0 for all x,y.
  Table (size 2): [[0,0],[0,0]]
  Properties: all products are 0. Satisfies x*y = z*w type equations.

=== HOW TO VERIFY AN EQUATION ON A MAGMA ===

Example A — LP FAILS E1 (do NOT use as CE):
  E1: "x = ((y * y) * z) * x"
  Leftmost leaf of RHS = y (not x). LP gives: x = y. FAILS for x≠y.
  STOP — LP is invalid CE. Do not proceed.

Example B — LP holds E1, but also holds E2 (still NOT a CE):
  E1: "x = x * ((x * x) * y)"  Leftmost leaf = x → LP gives x = x ✓
  E2: "x = x * (y * (z * (w * x)))"  Leftmost leaf = x → LP gives x = x ✓
  LP satisfies E2 too — LP is NOT a counterexample. Look for another magma.

Example C — RP holds E1, but also holds E2 (still NOT a CE):
  E1: "x = ((y * y) * z) * x"  Rightmost leaf = x → RP gives x = x ✓
  E2: "x = y * (y * (z * (w * x)))"  Rightmost leaf = x → RP gives x = x ✓
  RP satisfies both. RP is NOT a counterexample.
  → Neither LP nor RP can be CE here. Check XOR, Z3A, C0. If all fail → TRUE.

Example D — RP is a valid CE:
  E1: "x = y * x"  Rightmost leaf of RHS = x → RP gives x = x ✓ E1 holds.
  E2: "x = x * y"  Leftmost leaf of RHS = x; but on RP: x*y = y, not x.
  Take x=0, y=1: RP gives 0*1 = 1 ≠ 0. E2 FAILS. ✓ RP is valid CE → FALSE.

=== KEY PROOF STRATEGIES FOR TRUE ===

Strategy 1 — ABSORPTION: If E1 forces x*y = x (left-absorb) or x*y = y (right-
absorb), then ALL terms collapse to a single variable. E1 then implies almost
any E2. Check: substitute specific values into E1 to derive x*y=x or x*y=y.

Strategy 2 — VARIABLE SUBSTITUTION: Set free variables equal.
  E.g., from "x = x * (y * (z * (y * y)))", set y=x and z=x to get
  x = x * (x * (x * (x * x))). Then show this forces x*y=x for all y.

Strategy 3 — TRANSITIVITY: If E1 implies some intermediate law L, and L implies
E2, then E1 implies E2. Common intermediate: left-absorb, right-absorb,
idempotency (x*x=x), commutativity (x*y=y*x).

Strategy 4 — TERM REWRITING: If E1 says x = f(x,y,...), substitute this
identity into E2 systematically. If every occurrence of "x" in E2 can be
replaced using E1 to make E2 hold, then TRUE.

=== KEY PATTERNS (from verified database) ===

PATTERN A — Strong equations imply everything:
  Equations with structure like "x = y * (... complex term with x,y,z ...)"
  where the free variable z makes the equation very restrictive often imply
  simpler equations. If you verify such E1 on LP and LP FAILS E1, and on RP
  and RP FAILS E1, and on XOR and XOR FAILS E1: E1 is likely very strong.
  If no small magma satisfies E1 except trivial ones, E1 probably implies E2.

=== FALSE COUNTEREXAMPLE COOKBOOK ===

When E1 is a "weak" equation (LP or RP satisfies it), use that magma as CE:

LP satisfies these common forms:
  x = x * (... anything ...) — because x*(...)=x on LP
  x = (... x-rooted term ...) * (... anything ...) — if inner term has x as leftmost
  Example: "x = x * ((y * z) * (x * w))" → LP satisfies, RHS=x*(y*x)=x*x=x ✓

RP satisfies:
  x = y * x — right-absorb law
  x = (... anything ...) * x — because (...)*x = x on RP

XOR satisfies:
  Equations balanced under mod-2 arithmetic
  x = y * ((y * (y * x)) * y) — check: y*(y*x)=y+y+x=x, y*x=y+x, y*(y+x)=x ✓

TO DISPROVE E1 → E2 (i.e., show FALSE):
  1. Pick a candidate CE magma (LP, RP, XOR, Z3A, C0)
  2. Substitute the magma's operation into E1. Check ALL variable combinations.
  3. If E1 holds on all inputs → magma satisfies E1 ✓
  4. Find ONE input combination where E2 fails → magma refutes E2 ✓
  5. Present the magma and the failing assignment.

=== COMMON MISTAKES TO AVOID ===

MISTAKE 1: Using LP/RP as CE without checking leftmost/rightmost leaf first.
  "x = ((y*y)*z)*x": leftmost leaf of RHS = y, not x. LP gives x=y → FAILS E1!
  "x = x*(y*(z*(x*y)))": rightmost leaf = y, not x. RP gives x=y → FAILS E1!
  If leaf ≠ x, that magma CANNOT be CE. Do this check FIRST, before anything.

MISTAKE 2: Defaulting to FALSE when no CE is obvious.
  If you cannot find a valid small CE, the answer is likely TRUE.
  Many TRUE implications have proofs via substitution + absorption chains.

MISTAKE 3: Confusing left-absorb (x*y=x) with right-absorb (x*y=y).
  These are different! LP gives x*y=x. RP gives x*y=y.
  Check your magma table: LP = [[0,0],[1,1]], RP = [[0,1],[0,1]].

=== DECISION PROCEDURE ===
Given "Does E1 imply E2?":
1. Try to prove TRUE first:
   a. Substitute variables in E1 to derive simpler identities.
   b. Check if E1 forces absorption (x*y=x or x*y=y). If so → TRUE.
   c. Check if E2 follows from E1 by term rewriting.
2. If proof attempt fails, try to disprove (find CE):
   a. LP check: find leftmost leaf of E1's RHS.
      leaf ≠ x → skip LP entirely.
      leaf = x → verify E1 fully, then check E2. If E2 also holds → LP not CE.
   b. RP check: find rightmost leaf of E1's RHS.
      leaf ≠ x → skip RP entirely.
      leaf = x → verify E1 fully, then check E2. If E2 also holds → RP not CE.
   c. XOR check: compute E1 using mod-2 arithmetic. If E1 holds, check E2.
   d. Z3A check: mod-3 arithmetic. Same.
   e. C0 check: all products = 0. E1 becomes x=0, holds only for trivial eq.
3. ALWAYS verify E1 holds on your chosen CE before writing COUNTEREXAMPLE.
4. If NO valid CE found from all 5 checks → answer TRUE with a proof attempt.
  Do NOT output FALSE without a verified CE — lean TRUE when in doubt.
