MAGMA IMPLICATION GUIDE
E1,E2 are equations over magmas with one binary operation *.
Question: does every magma satisfying E1 also satisfy E2?

CRITICAL CALIBRATION
- Stage 1 evaluation is balanced 50/50. Do NOT default to FALSE from prior odds.
- Say TRUE only from an explicit proof family below.
- Say FALSE only from an actual counterexample pattern.

1. SAME LAW -> TRUE
If E1 and E2 are the same equation up to consistent variable renaming, TRUE.

2. SINGLETON -> TRUE
If one side of E1 is a bare variable absent from the other side, E1 forces x=y,
so every E2 holds.
Examples: x=y*z, x=(y*z)*w.
Non-example: x=y*(x*z).

3. TRIVIAL TARGET -> TRUE
If E2 is x=x, TRUE.

4. LEFT-PROJECTION FAMILY -> TRUE/FALSE BY LEFTMOST VARIABLE
Graph-backed members of the class of x=x*y include:
  x=x*y
  x=x*(y*z)
  x=x*(y*(x*z))
  x=x*(y*(z*y))
  x=x*((y*z)*w)
and variable-renamings of these shapes.
These force a*b=a.
Then E1=>E2 iff the leftmost variable on both sides of E2 is the same.

5. RIGHT-PROJECTION FAMILY -> TRUE/FALSE BY RIGHTMOST VARIABLE
Graph-backed members of the class of x=y*x include:
  x=y*x
  x=(y*z)*x
  x=((y*y)*z)*x
  x=(y*(y*(y*z)))*x
  x=((y*z)*w)*x
and variable-renamings of these shapes.
These force a*b=b.
Then E1=>E2 iff the rightmost variable on both sides of E2 is the same.

6. CONSTANT-PRODUCT FAMILY -> TRUE/FALSE BY WHETHER E2 HAS PRODUCTS
Graph-backed members of the class of x*y=z*w include:
  x*y=z*w
  x*y=y*z
  x*y=z*y
  x*x=y*z
  x*y=(y*x)*z
  x*y=(y*z)*x
and variable-renamings of these shapes.
These force all products to be equal to one constant c.
Consequences:
  if both sides of E2 contain * -> TRUE
  if E2 is x=x -> TRUE
  if one side of E2 is bare and the other contains * -> FALSE via constant model
  if E2 is x=y -> not automatic; only singleton gives that

7. SQUARE-BRIDGE FAMILY -> A SMALL EXTRA TRUE LANE
Graph-backed source members include:
  x=y*(x*x)
  x=(y*x)*(y*x)
  x=(y*(y*z))*(w*x)
  x=(y*(y*x))*(z*x)
  x=(x*x)*y
  x=(x*y)*(z*z)
and variable-renamings of these shapes.
These are weaker than full projection families. Use them only for graph-backed target shapes below.
Safe TRUE targets from this lane include:
  x=((x*y)*x)*(z*x)
  x=y*(z*(w*(u*x)))
  x*y=z*(y*(w*y))
  x*x=x*((y*z)*w)
If E1 matches one of the listed source shapes and E2 matches one of the listed target shapes up to renaming, TRUE.

8. FALSE VIA SMALL COUNTEREXAMPLE MODELS
Always check E1 first. A model is useful only if it satisfies E1 for all assignments.

LEFT PROJECTION: a*b=a
  E holds iff leftmost variables match on both sides.

RIGHT PROJECTION: a*b=b
  E holds iff rightmost variables match on both sides.

CONSTANT: a*b=0
  E holds iff both sides contain *, or both sides are the same bare variable.

XOR ON {0,1}: a*b=a+b mod 2
  E holds iff each variable has the same parity on both sides.

If any one of these models satisfies E1 but not E2, answer FALSE and name it.

9. SAFE FALSE FILTERS
- If E2 is x=y and E1 is not singleton-forcing, FALSE.
- If E1 is balanced and E2 is unbalanced, and E1 is not already in singleton,
  projection, constant-product, or square-bridge families, FALSE.

10. PROOF TOOLS
- Substitution: from x=x*y, setting y=x gives x=x*x.
- Duality: reverse every product. E1=>E2 iff dual(E1)=>dual(E2).
- For TRUE, reduce to one of these normal forms:
  same law / singleton / left projection / right projection / all products equal / square bridge.

11. HARD TRUE EXAMPLES TO REMEMBER
- x=((y*y)*z)*x => x=x*(((y*x)*x)*x) is TRUE.
  Reason: E1 is in the right-projection family, and both sides of E2 end in x.
- x*y=(y*x)*z => x*y=z*((w*y)*y) is TRUE.
  Reason: E1 is in the constant-product family, and both sides of E2 contain *.
- x=x*(y*(x*z)) => x=((x*y)*(z*w))*u is TRUE.
  Reason: E1 is in the left-projection family, and both sides of E2 start with x.
- x=(y*x)*(y*x) => x=((x*y)*x)*(z*x) is TRUE.
  Reason: E1 is in the graph-backed square-bridge family, and E2 is one of its known target shapes.

12. WARNINGS
- Do NOT use "most hard problems are FALSE". This repo's official eval target is balanced.
- x=x*(x*y) is NOT the same as x=x*y.
- Similar-looking equations may belong to different graph classes.
- The square-bridge lane is narrow. Do NOT generalize it to every equation ending in x.
- Counterexample first for FALSE; explicit family first for TRUE.
