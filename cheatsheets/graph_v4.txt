MAGMA IMPLICATION CHEATSHEET
E1,E2: equations over magmas (set with binary op *).
Question: Does every magma satisfying E1 also satisfy E2?

====== DECISION PROCEDURE ======

STEP 1: SINGLETON â†’ TRUE
If one side of E1 is a bare variable NOT appearing on the other side,
then E1 forces all elements equal (singleton magma). Any E2 holds. â†’ TRUE.
  Ex: x = y*(z*w) â†’ x not in RHS â†’ singleton â†’ TRUE
  Ex: x = y*(x*z) â†’ x IS in RHS â†’ NOT singleton â†’ continue

STEP 2: TRIVIAL â†’ TRUE
If E2 is x=x â†’ TRUE (every magma satisfies x=x).

STEP 3: LEFT PROJECTION TEST
Check if E1 has form x = x * EXPR where:
 (a) The FIRST variable reading left-to-right in EXPR is NOT x, AND
 (b) That first variable appears EXACTLY ONCE in EXPR.

YES examples (class 4 - forces left projection a*b=a):
  x=x*y           EXPR=y, first var=yâ‰ x, y appears 1x âś“
  x=x*(y*z)       EXPR=(y*z), first=yâ‰ x, 1x âś“
  x=x*(y*(x*z))   EXPR=(y*(x*z)), first=yâ‰ x, 1x âś“
  x=x*((y*z)*w)   EXPR=((y*z)*w), first=yâ‰ x, 1x âś“

NO examples (NOT class 4):
  x=x*(x*y)       first var=x â†’ FAIL (this is class 9, much weaker)
  x=x*(y*(z*y))   first=y but y appears 2x â†’ FAIL
  x=x*((x*y)*z)   first var=x â†’ FAIL

If E1 IS class 4 (left projection):
  Under a*b=a, every expression evaluates to its LEFTMOST variable.
  E1âŠ¨E2 iff leftmost var of LHS(E2) = leftmost var of RHS(E2).
  â†’ Same leftmost var â†’ TRUE    â†’ Different â†’ FALSE
  Ex: E2 = x*(y*z) = x*y â†’ leftmost: x vs x â†’ TRUE
  Ex: E2 = x*y = y*x â†’ leftmost: x vs y â†’ FALSE

STEP 4: RIGHT PROJECTION TEST
Check if E1 has form x = EXPR * x where:
 (a) The FIRST variable reading left-to-right in EXPR is NOT x, AND
 (b) That first variable appears EXACTLY ONCE in EXPR.

YES examples (class 5 - forces right projection a*b=b):
  x=y*x           EXPR=y, first=yâ‰ x, 1x âś“
  x=(y*z)*x       EXPR=(y*z), first=yâ‰ x, 1x âś“
  x=(y*(z*x))*x   EXPR=(y*(z*x)), first=yâ‰ x, 1x âś“
  x=((y*z)*w)*x   EXPR=((y*z)*w), first=yâ‰ x, 1x âś“

If E1 IS class 5 (right projection):
  Under a*b=b, every expression evaluates to its RIGHTMOST variable.
  E1âŠ¨E2 iff rightmost var of LHS(E2) = rightmost var of RHS(E2).
  â†’ Same rightmost var â†’ TRUE    â†’ Different â†’ FALSE

STEP 5: COUNTEREXAMPLE MAGMAS â†’ FALSE
A single magma satisfying E1 but NOT E2 proves FALSE.
Test these models on BOTH E1 and E2. If model satisfies E1 but not E2 â†’ FALSE.

(A) LEFT PROJECTION: a*b = a
    Any expression reduces to its leftmost variable.
    E holds iff leftmost vars match on both sides.
    Most equations with a bare var on LHS are satisfied (leftmost=x both sides).

(B) RIGHT PROJECTION: a*b = b
    Any expression reduces to its rightmost variable.
    E holds iff rightmost vars match on both sides.

(C) CONSTANT: a*b = c (fixed constant, all products equal)
    Any expression containing * evaluates to c.
    A bare variable evaluates to itself.
    E holds iff: both sides have * (â†’both=c, equal),
    OR both sides are the same bare variable,
    OR one side is bare var, other side IS that same var.
    E fails if: one side has * (â†’c) and other is a bare var (unless that var=c).

(D) VARIABLE PARITY (XOR on {0,1}, a*b = aâŠ•b):
    For each variable, count its occurrences on each side of E.
    E holds iff every variable has same parity (odd/even count) on both sides.
    Ex: (x*y)*x = y â†’ LHS: x appears 2(even), y appears 1(odd)
                       RHS: x appears 0(even), y appears 1(odd) â†’ same parity â†’ holds
    Ex: x*y = y*(x*z) â†’ LHS z:0, RHS z:1 â†’ different parity â†’ fails

Apply: if any model (A)-(D) satisfies E1 but NOT E2 â†’ FALSE.
Use that model as your counterexample.

STEP 6: STRUCTURAL FILTERS â†’ FALSE
(a) BALANCE: Count * on each side of E1 and E2.
    If E1 is balanced (same * count both sides) and E2 is NOT â†’ FALSE.
    Reason: any model (left proj, right proj, constant, XOR) satisfying
    a balanced equation preserves balance.

(b) E2 = x=y (different bare variables, no *):
    Only singleton magmas satisfy this. If E1 is not singleton â†’ FALSE.

STEP 7: SUBSTITUTION â†’ TRUE
Try substituting variables in E1 to derive E2 or consequences:
  From E1, substitute yâ†’x: get a specialized equation E1'.
  If E1' matches E2 directly â†’ TRUE.
  If E1' can be chained with E1 to produce E2 â†’ TRUE.
  Ex: E1 = x=x*y. Set y=x: x=x*x (idempotent).
      Set y=z*w: x=x*(z*w). These are consequences of E1.

STEP 8: DUALITY
The dual of equation E: swap all * arguments (a*b becomes b*a).
E1âŠ¨E2 iff dual(E1)âŠ¨dual(E2).
Left projection â†” right projection under duality.

STEP 9: STRENGTH ORDERING
Stronger equations (fewer satisfying magmas) imply more.
Strength indicators:
  - Bare variable on one side â†’ very strong
  - More distinct variables â†’ generally stronger
  - x=y is the strongest (forces singleton)
  - x=x is the weakest (satisfied by everything)
  Chain: x=y â†’ x=x*y â†’ x=x*(x*y) â†’ x*x=x â†’ x=x

STEP 10: FINAL REASONING
If no rule resolved it:
â€˘ Try to find a 2-element magma ({0,1}, 4 entries) satisfying E1 but not E2.
  There are only 16 possible binary operations on {0,1}. Common ones:
  AND(0,0,0,1) OR(0,1,1,1) NAND(1,1,1,0) NOR(1,0,0,0) IMPL(1,1,0,1)
  Format: (0*0, 0*1, 1*0, 1*1)
  LP=(0,0,1,1) RP=(0,1,0,1) CONST0=(0,0,0,0)
â€˘ Try algebraic derivation of E2 from E1 using substitution and rewriting.
â€˘ If E1 is "obviously stronger" (simpler, bare vars, more constraints) â†’ TRUE.
â€˘ If E1 looks complex but E2 imposes different structural constraints â†’ FALSE.

====== CONSTANT-FORCING HINT ======
If E1 has * on both sides, 3+ distinct variables, and is NOT satisfied
by LP or RP, it MIGHT force constant operation (all products equal one value).
If constant satisfies BOTH E1 and E2 â†’ consider TRUE. But be cautious:
many such equations do NOT force constant. Strong evidence:
  - E1 equates products with DISJOINT variables: x*y=z*w (definitely constant)
  - E1 equates with rearranged vars: x*y=(y*x)*z, x*y=y*((x*y)*z)
If uncertain whether E1 forces constant, try a 2-element magma search.

====== CRITICAL WARNINGS ======
- Both TRUE and FALSE are equally likely. Do NOT default to either.
- "Similar looking" equations may NOT imply each other.
- x=x*f(x,y) and x=x*(x*f(x,y)) are VERY different in strength.
  The first (class 4) forces left projection. The second (class 9) does NOT.
- An equation with more * is not necessarily stronger.
- Always try at least one counterexample magma before concluding TRUE.
- If E1 has * on both sides and E2 has a bare var on one side,
  check constant model: if both sides of E1 have * â†’ constant satisfies E1.
  If E2 has bare var vs expression with * â†’ constant fails E2 â†’ FALSE.
