MAGMA IMPLICATION CHEATSHEET

Read * as the binary operation.
Mandatory order:
instant -> singleton -> specialization -> duality -> multiplicity -> stock magmas -> linear magmas -> hard TRUE templates -> verdict.

Answer TRUE only from an explicit path.
Answer FALSE only from an explicit witness.

1. INSTANT
- Eq1 = Eq2 -> TRUE.
- Eq2 is x=x -> TRUE.
- Eq1 is x=x and Eq2 is not -> FALSE.
- If Eq1 is x=y or any singleton-forcing law, then Eq1 implies every Eq2.

2. SINGLETON / E2-TYPE
- Exact offline fact: 1496 of 4694 equations are in the singleton/E2 class.
- Safe strong trigger: LHS-vars and RHS-vars are disjoint and at least one side contains *.
  Then the law behaves like x=y: every model collapses to one element.
- High-yield singleton shapes:
  x = t(vars not x)
  t(vars not x) = x
  x*x = y*z
  x*y = z*w
- Use it asymmetrically:
  if Eq1 is singleton-forcing -> TRUE.
  if Eq2 is singleton-forcing and Eq1 is clearly not -> usually FALSE.
- Worked TRUE:
  Eq1: x = y*(z*w)
  Eq2: x = y*(z*y)
  Eq1 is singleton-forcing, so Eq2 follows automatically.

3. SPECIALIZATION: main TRUE mechanism
- Mined proof data: 6661 of 10657 direct proofs (62.5%) are specialization.
- Meaning: Eq2 is obtained from Eq1 by replacing variables of Eq1 with variables or terms, consistently.
- Consistent means the same variable must map to the same term every time.
- Repeated variables in Eq2 are strong TRUE evidence: they often come from identifying variables in Eq1.
- Worked examples:
  A. Eq1: x*y = x*z
     Eq2: x*x = x*y
     substitution: y<-x, z<-y
     so Eq2 is a direct instance of Eq1. TRUE.
  B. Eq1: x = y*(z*x)
     Eq2: x = y*(x*x)
     substitution: z<-x
     TRUE.
  C. Eq1: x = (x*y)*z
     Eq2: x = (x*x)*y
     substitution: y<-x, z<-y
     TRUE.
  D. Eq1: x = x*(y*z)
     Eq2: x = x*(y*x)
     substitution: z<-x
     TRUE.
  E. Eq1: x = (y*z)*x
     Eq2: x = (y*y)*x
     substitution: z<-y
     TRUE.
  F. Eq1: x = y*(y*z)
     Eq2: x = y*(y*y)
     substitution: z<-y
     TRUE.
- Constant and projection families are especially high-yield:
  constant -> constant: 1698 mined proofs
  singleton -> singleton: 1023 mined proofs
  right projection -> right projection: 479 mined proofs
  left projection -> left projection: 471 mined proofs
- Non-example:
  Eq1: x*y = x*z cannot directly yield x*y = y*z.
  The left x would have to become both x and y, which is inconsistent.

4. DUALITY
- dual(u*v) means reverse the product tree: dual(u*v) = dual(v) * dual(u).
- Eq1 => Eq2 iff dual(Eq1) => dual(Eq2).
- So if direct specialization is not obvious, dualize both equations and try again.
- Worked example:
  Eq1: x = (y*z)*x
  dual Eq1: x = x*(z*y)
  Eq2: x = (y*y)*x
  dual Eq2: x = x*(y*y)
  dual Eq2 is a specialization of dual Eq1 by z<-y.
  Therefore original Eq1 => Eq2 is TRUE.
- CRITICAL:
  if you would answer differently on a dual-swapped pair, you are wrong.
  Duality must preserve the verdict.

5. REWRITE / STRUCTURAL SIMPLIFICATION
- Mined proof data: 1807 proofs are structural simplification and 1661 are short rewrites.
- Use this when Eq1 and Eq2 keep the same variable support and Eq1 looks deeper or stronger.
- Common situation: Eq1 collapses an outer frame, so the same frame still collapses after a variable is replaced by a more nested term.
- Rewrite examples:
  A. From x*y = x, set y<-y*z.
     Then x*(y*z) = x. TRUE.
  B. From x*y = y, first x*y = y.
     Reapply the same law to y*z.
     Then (x*y)*z = y*z = z. So (x*y)*z = z. TRUE.
  C. From x = x*(y*z), set z<-x.
     Then x = x*(y*x). TRUE.
  D. From x = (y*z)*x, set z<-y.
     Then x = (y*y)*x. TRUE.
- Projection-style collapse is extremely common:
  left anchored family:  x = x*(...)
  right anchored family: x = (...)*x
  Once the outside anchor matches, deeper nested interiors often still collapse.

6. HARD TRUE TEMPLATES
- These were repeatedly missed by the old cheatsheets. Do not default FALSE just because they look deep.
- Eq54-type self-application:
  source family: x = x*(y*(x*z))
  first moves: z<-x, then maybe y<-x.
  likely TRUE targets:
  x = x*(y*(x*x))
  x = x*(x*(x*z))
  x = x*(x*(x*x))
- Eq1576-type repeated y-block:
  source family: x = (y*z)*(y*(y*w))
  first moves: z<-y, w<-y, or both.
  repeated y usually signals a specialization path, not a counterexample.
- Eq2544-type absorption:
  source family: x = (y*((y*y)*y))*y
  treat m=(y*y)*y as one block.
  many TRUE targets keep the outer y ... y frame and simplify only the middle block.
- Constant-side hard TRUEs:
  if Eq1 forces x*x = x*(...), then variants with repeated variables on the right are often direct specializations.

7. BALANCED-VS-UNBALANCED REFUTATION
- Count occurrences of each variable on LHS and RHS.
- If Eq1 is balanced for every variable but Eq2 is unbalanced for some variable, then FALSE.
- Why: rewriting under a balanced law preserves multiplicity balance.
- Worked FALSE:
  Eq1: x*y = y*x
  counts: x and y each appear once on both sides.
  Eq2: x*y = x
  y disappears on the right.
  So Eq1 cannot imply Eq2. FALSE.
- This is a fast, reliable FALSE rule. Use it before long search.

8. HOW TO USE A SMALL MAGMA
- Need one witness magma where Eq1 holds for all valuations and Eq2 fails for some valuation.
- Evaluate both sides bottom-up.
- Table encoding:
  ab/cd means row0=[a,b], row1=[c,d].
  Example: left-zero 00/11 means 0*0=0, 0*1=0, 1*0=1, 1*1=1.
- Worked FALSE with const0 00/00:
  Eq1: x*y = y*x
  Both sides are always 0, so Eq1 holds.
  Eq2: x*y = x
  valuation x=1, y=0 gives LHS=0, RHS=1.
  Eq2 fails, so Eq1 does not imply Eq2. FALSE.
- Never claim FALSE unless you can name the table and the failing valuation.

9. STOCK SMALL MAGMAS
- These refute most easy and medium FALSE cases.
- const0 00/00
- const1 11/11
- left-zero 00/11
- right-zero 01/01
- xor 01/10
- and 00/01
- or 01/11
- nand 11/10
- implication-like 11/01
- left-and-not 01/00
- right-xor-and 00/10
- What they are good for:
  const0, const1: constant-side and total-collapse failures
  left-zero, right-zero: projection asymmetry
  xor, and, or, nand: commutative/idempotent traps
  implication-like, left-and-not, right-xor-and: noncommutative negatives missed by symmetric tables

10. LINEAR MAGMAS
- If stock tables miss, try x*y = a x + b y mod p for small primes, especially p=3.
- High-yield tables:
  012/201/120 = 2x+y mod 3
  021/102/210 = x+2y mod 3
  012/120/201 = x+y mod 3
  000/111/222 = left projection on 3
  012/012/012 = right projection on 3
  0123/1032/2301/3210 = Klein-4 style xor on 4
- Mined hard FALSE data: 67 sampled pairs were separated by linear magmas after stock tables failed.
- If stock and linear both miss, harder FALSE families exist offline:
  translation-invariant tables x*y = x + f(y-x)
  twisted constructions
  They matter for research, but are too low-frequency to spend many cheatsheet bytes here.

11. ANTI-COLLAPSE GUARDRAILS
- Not finding a counterexample in the stock tables is evidence for TRUE, not permission to guess FALSE.
- If Eq2 uses no new variables and looks like Eq1 with some variables identified, TRUE is structurally plausible.
- Deep self-application is not automatically FALSE:
  x*(x*x)
  x*(y*(x*z))
  (y*z)*(y*(y*w))
- Projection families are high-yield:
  x = x*(...) often implies deeper left-anchored targets.
  x = (...)*x often implies deeper right-anchored targets.
- The old failure mode was:
  looks hard -> answer FALSE.
  Reject that behavior.

12. VERDICT DISCIPLINE
- TRUE requires one of:
  exact equality
  Eq2 is x=x
  Eq1 singleton-forcing
  explicit specialization
  explicit dual-specialization
  short rewrite chain you can state
- FALSE requires one of:
  multiplicity refutation
  explicit stock magma witness
  explicit linear magma witness
- Two-pass rule:
  Pass 1: multiplicity + stock/linear magmas.
  Pass 2: if Pass 1 fails, mandatory specialization/duality/rewrite attempt.
- If neither side has a real argument yet, keep checking. Do not default to FALSE.

13. GRAPH-POSITION FROM FORMULA (mined stats, 1415-equation corpus)
ops = total * in formula. avg_implies by ops:
  ops=0  n=2    avg=2346  (x=x, x=y)
  ops=1  n=3    avg= 822  (x=x*y, x=y*x)
  ops=2  n=18   avg= 208  (x*x=y*z, x=y*(x*x))
  ops=3  n=123  avg=  40

  vars=2  avg=20   vars=3  avg=37   vars=4  avg=66
Lower ops + more distinct vars = stronger hub (higher out-degree).

14. NO-REPEATED-VARS RULE
If every variable appears exactly once total (both sides combined):
  avg_implies=203  (8.7x corpus mean of 23)  -> treat Eq1 as high-power hub.
Any repeated variable drops avg to 23.

15. OPS-ASYMMETRY RULE
asym=|ops_LHS - ops_RHS|:  asym=1 peaks at avg=105 implies.
asym=0 avg=50, asym=2 avg=23, asym=4 avg=20.
Eq1 with exactly one-side-more operator -> boost TRUE confidence.

16. ONE-VAR SINK RULE
Eq with only one distinct variable: avg_implies=2.7, avg_impby=2269.
  x*x=x*(x*(x*x))   Eq3253 impby=3029  x*x=((x*x)*x)*x  Eq4065 impby=3029
  x*x=x*(x*x)        Eq307  impby=2617  x*x=(x*x)*(x*x)  Eq3659 impby=2575
If Eq2 is one-var -> lean TRUE. If Eq1 is one-var -> lean FALSE.

17. HUB QUICK-SCORE for Eq1 (estimate out-degree)
+60 ops<=1  +40 ops=2  +20 ops=3  +20 vars=4  +10 vars=3
+30 no variable repeated anywhere  +50 LHS-vars and RHS-vars disjoint
-15 balanced (every var same count LHS as RHS)  -25 vars=1
Score>=70: strong hub, lean TRUE. Score<0: sink, lean FALSE.
