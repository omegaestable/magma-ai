STRUCTURAL WITNESS TESTS
═══════════════════════════════════════════════════════

For each test, determine whether the equation HOLDS by the structural rule below.
A witness SEPARATES E1 from E2 when E1 holds but E2 does NOT.

■ TEST 1 — LP (Left Projection):
Rule: An equation "A = B" holds on LP iff the LEFTMOST variable in A equals the LEFTMOST variable in B.
How: scan each side left-to-right; the first single letter (x, y, z, w, u, v) you encounter is the leftmost variable of that side.
Example: "x = ((y * z) * x) * w" → leftmost of LHS is x, leftmost of RHS is y → x ≠ y → FAILS LP.
Example: "x = (x * y) * z" → leftmost of LHS is x, leftmost of RHS is x → x = x → HOLDS on LP.
Example: "x * y = (y * x) * z" → leftmost of LHS is x, leftmost of RHS is y → x ≠ y → FAILS LP.

■ TEST 2 — RP (Right Projection):
Rule: An equation "A = B" holds on RP iff the RIGHTMOST variable in A equals the RIGHTMOST variable in B.
How: scan each side right-to-left; the first single letter you encounter is the rightmost variable.
Example: "x = y * (z * x)" → rightmost of LHS is x, rightmost of RHS is x → HOLDS on RP.
Example: "x = (x * y) * z" → rightmost of LHS is x, rightmost of RHS is z → x ≠ z → FAILS RP.

■ TEST 3 — C0 (Constant Zero):
Rule: An equation "A = B" holds on C0 iff EITHER:
  (a) Both A and B contain at least one * operator, OR
  (b) Neither side contains * and both sides are the same single variable.
If one side has * and the other is a bare variable (no *), C0 FAILS.
Example: "x = (y * z) * w" → LHS has no *, RHS has * → one bare, one compound → FAILS C0.
Example: "x * y = (z * w) * x" → both sides have * → HOLDS on C0.
Example: "x = x" → both sides are bare x, same variable → HOLDS on C0.

■ TEST 4 — XOR (Addition mod 2):
Rule: An equation "A = B" holds on XOR iff for EVERY variable, its count on the left side and right side have the same parity (both even or both odd).
How: count how many times each variable appears as a leaf in A and in B. Compare parities.
Example: "x = (x * y) * y" → LHS counts: x=1. RHS counts: x=1, y=2. Check: x is 1≡1 (same parity ✓), y is 0≡2 (both even ✓) → HOLDS on XOR.
Example: "x = y * (z * x)" → LHS: x=1. RHS: y=1, z=1, x=1. Check: x is 1≡1 ✓, y is 0≡1 (different parity ✗) → FAILS XOR.

■ TEST 5 — XNOR:
Rule: An equation holds on XNOR iff BOTH conditions are met:
  (a) The XOR test passes (all variable count parities match), AND
  (b) The count of * operators on the left side and right side have the same parity.
Example: "x = (x * y) * y" → XOR passes (from above). Stars: LHS has 0 stars, RHS has 2 stars. 0 and 2 are both even → HOLDS on XNOR.
Example: "x * y = (x * y) * z" → XOR: LHS x=1,y=1; RHS x=1,y=1,z=1. z is 0≡1 → different → FAILS XOR → FAILS XNOR.

■ TEST 6 — AND/OR (both use the same rule):
Rule: An equation "A = B" holds on AND (and identically on OR) iff:
  - If BOTH sides contain *, the SET of distinct variables in A equals the SET in B.
  - If NEITHER side contains *, both must be the same single variable.
  - If one side has * and the other is a bare variable v, the * side must contain ONLY variable v.
Example: "x * y = (y * x) * y" → both have *. Vars in LHS: {x,y}. Vars in RHS: {y,x} = {x,y}. Same set → HOLDS.
Example: "x = (x * y) * z" → LHS bare variable x. RHS has * and uses {x,y,z}. {x,y,z} ≠ {x} → FAILS.
Example: "x = x * x" → LHS is bare x. RHS has * and uses only {x}. {x} = {x} → HOLDS.

■ TEST 7 — Z3A (Addition mod 3):
Rule: An equation "A = B" holds on Z3A iff for EVERY variable, its count in A mod 3 equals its count in B mod 3.
Example: "x = (x * x) * x" → LHS: x=1. RHS: x=3. 1 mod 3 = 1, 3 mod 3 = 0 → 1 ≠ 0 → FAILS Z3A.
Example: "x = (y * (y * y)) * x" → LHS: x=1. RHS: y=3, x=1. x: 1≡1 ✓. y: 0 mod 3 = 0, 3 mod 3 = 0 → 0≡0 ✓ → HOLDS on Z3A.

IMPORTANT REMINDERS:
- These are purely TEXT-BASED checks. No Cayley tables or number crunching needed.
- LP and RP: just find the first/last variable on each side of =.
- C0: just check whether each side contains a * symbol.
- XOR: count variable appearances and compare parities.
- XNOR: XOR rule plus compare star-count parities.
- AND/OR: compare the sets of variables (ignoring how many times each appears).
- Z3A: count variable appearances and compare mod 3.
- Work through each test methodically. Do not skip tests or guess.
