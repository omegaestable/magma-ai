MAGMA IMPLICATION CHEATSHEET
Given E1,E2: does every magma satisfying E1 also satisfy E2?

STEP 1: PARSE
Strip spaces. Identify LHS/RHS of each equation (split on =).
Count * per side. List all variables per side.

STEP 2: SINGLETON TEST → TRUE
If E1 has one side that is a bare variable NOT appearing on the other side:
  E1 forces the magma to be a singleton. TRUE for any E2.
  Ex: "x=y*(z*w)" → x is bare, x not on RHS → singleton → TRUE.
  Ex: "x=y*(x*z)" → x IS on RHS → NOT singleton → continue.

STEP 3: TRIVIAL TARGET → TRUE
If E2 is "x=x" → TRUE (every equation implies x=x).

STEP 4: IDENTIFY E1 CLASS
(a) LEFT-ABSORPTIVE: E1 has form x=x*EXPR where EXPR contains some variable
    other than x. Examples: x=x*y, x=x*(y*z), x=x*(y*(x*z)), x=x*((y*z)*w).
    This forces LEFT PROJECTION: a*b=a for all a,b.
(b) RIGHT-ABSORPTIVE: E1 has form x=EXPR*x where EXPR contains some variable
    other than x. Examples: x=y*x, x=(y*z)*x, x=((y*z)*w)*x.
    This forces RIGHT PROJECTION: a*b=b for all a,b.
(c) CONSTANT-FORCING: E1 forces all products equal. This happens when E1 has
    * on both sides and uses enough free variables to constrain all products
    to be equal. Key examples from this class:
    x*y=z*w, x*y=y*z, x*y=z*y, x*x=y*z, x*y=(y*z)*w, x*y=z*(w*u),
    x*y=(z*w)*u, and hundreds of similar forms where both sides have *
    and at least 3 distinct variables appear total.
    This forces CONSTANT: a*b=c for some fixed c.

STEP 5: USE E1 CLASS FOR IMPLICATION
(a) If E1 is LEFT-ABSORPTIVE (Step 4a):
    E1⊨E2 iff E2 is satisfied by left projection (a*b=a).
    TEST: evaluate both sides of E2 under a*b=a.
    Under left projection: any expression reduces to its leftmost variable.
    So E2 holds iff the leftmost variable is the SAME on both sides of E2.
    Same leftmost var → TRUE. Different leftmost var → FALSE.

(b) If E1 is RIGHT-ABSORPTIVE (Step 4b):
    E1⊨E2 iff E2 is satisfied by right projection (a*b=b).
    TEST: Under right projection, any expression reduces to its rightmost variable.
    E2 holds iff the rightmost variable is the SAME on both sides of E2.
    Same rightmost var → TRUE. Different rightmost var → FALSE.

(c) If E1 is CONSTANT-FORCING (Step 4c):
    E1⊨E2 iff E2 is satisfied by the constant operation (a*b=c for fixed c).
    TEST: Under constant operation, a bare variable evaluates to itself,
    any expression with * evaluates to c.
    - Both sides of E2 have * → both evaluate to c → TRUE.
    - One side bare variable, other has * → variable vs c → FALSE.
    - Both sides bare same variable → TRUE. Different bare vars → FALSE.

STEP 6: STRUCTURAL FILTERS → FALSE
(a) BALANCE TEST: Count * on each side of E1 and E2.
    If E1 is balanced (same * count on both sides) and E2 is NOT balanced,
    AND E1 does NOT force singleton → FALSE.
    Reason: balanced equations are satisfied by a*b=constant, which preserves
    the balanced property under rewriting.

(b) SINGLETON TARGET: If E2 is "x=y" (two different bare variables)
    and E1 is NOT singleton-forcing → FALSE.

STEP 7: COUNTEREXAMPLE MAGMAS → FALSE
Try these magmas. If one satisfies E1 but NOT E2, then FALSE.

LEFT_PROJ: a*b=a. Satisfies E iff leftmost var is same on both sides.
RIGHT_PROJ: a*b=b. Satisfies E iff rightmost var is same on both sides.
CONST: a*b=0. Satisfies E iff both sides have at least one * (or both are same bare var).
XOR on {0,1}: a*b=(a+b)mod2. Satisfies E iff variable parity (count mod 2)
  is same for each variable on both sides. x*x=y*y → x:2,y:2 each side ✓.
SUBTRACTION mod 3: a*b=(a-b)mod3.

If E1 is satisfied by LEFT_PROJ but E2 is NOT → FALSE.
If E1 is satisfied by RIGHT_PROJ but E2 is NOT → FALSE.
If E1 is satisfied by CONST but E2 is NOT → FALSE.
If E1 is satisfied by XOR but E2 is NOT → FALSE.

STEP 8: DUALITY PRINCIPLE
Dual of equation E: reverse all * (a*b→b*a throughout).
E1⊨E2 ⟺ dual(E1)⊨dual(E2).
If stuck, try checking the dual problem.

STEP 9: ALGEBRAIC REASONING
If E1 and E2 are not resolved by the above steps:
(a) Try direct derivation: substitute variables in E1 to derive E2.
    This proves TRUE.
(b) Try to find a small magma (2-4 elements) satisfying E1 but not E2.
    This proves FALSE.
(c) Consider: does E1 imply idempotency (x*x=x)? Many equations do.
    If E1 implies x=x*x and E2 requires non-idempotent behavior → FALSE.

STEP 10: KEY IMPLICATION CHAINS
These chains go from stronger (left) to weaker (right):
x=y → x=x*y → x=x*(x*y) → x=x*(x*(x*y)) → x=x*x → x=x
x=y → x=y*x → x=(y*z)*x → x=((y*z)*w)*x → x=x*x → x=x
x*y=z*w → x*y=z*y, x*y=y*z, x*x=y*y → x=x
If E1 is at or left of E2 in a chain → TRUE.
If E2 is to the left of E1 → FALSE (weaker cannot imply stronger).

STEP 11: FINAL DECISION
If no rule above resolves the case:
- Try algebraic derivation of E2 from E1.
- Try a small counterexample magma.
- Consider which equation imposes more constraints (fewer models).
- More constrained E1 (fewer models) is more likely to imply E2.
- An equation with a bare variable on one side is generally stronger.
- An equation using more distinct variables is generally stronger.

COMMON PITFALLS:
• "Similar looking" equations may NOT imply each other.
• More * operations does NOT mean stronger equation.
• Both TRUE and FALSE are common outcomes — do not default to either.
• Always check counterexample magmas before concluding TRUE.
• x=x*f and x=x*(x*f) are very different in strength.
