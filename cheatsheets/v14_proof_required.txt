=== CORE POLICY ===

To output VERDICT: TRUE, you need one of these:
1. An explicit derivation from E1 to E2 by substitution / rewriting.
2. An explicit derivation that E1 forces a very strong law (singleton, left-absorption,
   right-absorption, constant law, etc.) and then E2 follows from that law.
3. A correct finite-magma map argument (injective/surjective/bijective) with the map named
   and the deduction actually explained.

If all you have is:
- "E1 is restrictive",
- "LP/RP/XOR all fail E1",
- "I do not see a counterexample",
- "this feels Boolean / trivial / projection-like",
then you DO NOT have a proof of TRUE.

=== PAPER-BACKED QUICK CHECKS (DO THESE EARLY) ===

CHECK A — SINGLETON TARGET:
If E2 has the form x = t(y,z,w,...) and x does NOT appear anywhere in that RHS,
then E2 is equivalent to the singleton law.
Important: this does NOT make the implication TRUE by itself.
To answer TRUE, you must prove E1 also forces singleton / triviality.

CHECK B — VARIABLE MULTIPLICITY INVARIANT:
Count how many times each variable occurs on each side.
If E1 has matching multiplicities on its two sides, but E2 does not,
then E1 cannot imply E2. VERDICT: FALSE.
Example: commutative / associative style balanced laws preserve multiplicities.

CHECK C — LP / RP LEAF INVARIANTS (GENERAL FORM, NOT JUST x = RHS):
On LP, every term evaluates to its LEFTMOST leaf variable.
On RP, every term evaluates to its RIGHTMOST leaf variable.
So:
- If the two sides of E1 have the same leftmost leaf, LP satisfies E1.
- If the two sides of E2 have different leftmost leaves, LP refutes E2.
Then LP is a valid counterexample, so VERDICT: FALSE.
Likewise with rightmost leaves and RP.

CHECK D — DO NOT OVERCLAIM BOOLEAN-GROUP / XOR COLLAPSE:
Only claim "E1 forces abelian group of exponent 2" if you actually derived both:
- commutativity (or an equivalent symmetric law), and
- exponent-2 behavior / parity collapse.
Failure of LP/RP/AND/OR is NOT evidence for Boolean-group collapse.

CHECK E — ORDER / SHAPE OBSTRUCTIONS:
If E1 preserves some obvious syntactic invariant (leaf side, multiplicities, or a canonized
normal-form pattern) that E2 breaks, then the implication is FALSE.
Use this as negative evidence before trying to proclaim TRUE.

=== MANDATORY COUNTEREXAMPLE VERIFICATION PROTOCOL ===

Before claiming any magma M is a counterexample for "E1 implies E2":
1. Verify E1 holds on ALL inputs of M.
2. Verify E2 fails on SOME explicit input.
Only then is M a valid counterexample.

If your candidate fails E1, discard it immediately.
If your candidate satisfies E2 too, discard it immediately.

=== CANONICAL WITNESS LIBRARY ===

LP (Left Projection): x*y = x.
  Table (size 2): [[0,0],[1,1]]
  FAST RULE: every term = its leftmost leaf.

RP (Right Projection): x*y = y.
  Table (size 2): [[0,1],[0,1]]
  FAST RULE: every term = its rightmost leaf.

C0 (Constant zero): x*y = 0.
  Table (size 2): [[0,0],[0,0]]
  FAST RULE: every non-variable composite term evaluates to 0.
  Be careful if a whole side is just a lone variable like x.

XOR / Z2 addition: x*y = x+y mod 2.
  Table (size 2): [[0,1],[1,0]]
  Properties: associative, commutative, x*x=0.

Z3A / Z3 addition: x*y = x+y mod 3.
  Table (size 3): [[0,1,2],[1,2,0],[2,0,1]]
  Properties: associative, commutative.

AND on {0,1}: x*y = x AND y.
  Table: [[0,0],[0,1]]

OR on {0,1}: x*y = x OR y.
  Table: [[0,1],[1,1]]

NAND on {0,1}: x*y = 1 - x*y.
  Table: [[1,1],[1,0]]

XNOR / affine complement on {0,1}: x*y = 1-(x+y mod 2).
  Table: [[1,0],[0,1]]

These extra 2-element magmas are useful because hard FALSE cases often survive the basic
LP/RP/XOR tests but still fail in a tiny Boolean-style model.

=== HOW TO USE LP / RP CORRECTLY ===

For any term:
- LP: ignore all right children; keep following left children to the final leaf.
- RP: ignore all left children; keep following right children to the final leaf.

Examples:
  ((y*y)*z)*x
    LP -> y
    RP -> x

  y*(z*(y*(z*x)))
    LP -> y
    RP -> x

  (y*((y*x)*x))*y
    LP -> y
    RP -> y

This general test works even when the law is not written as x = RHS.

=== SAFE TRUE PATTERNS ===

PATTERN 1 — RHS omits x entirely:
If a law has the form x = f(y,z,w,...) with no x on the RHS, then that law is singleton-equivalent.
This is a real paper-backed rewrite fact.
But do NOT transfer this to E1 unless E1 itself has that shape or you derived it.

PATTERN 2 — Explicit left-absorptive schema:
If you can derive x = x * F(x,y,z,...) for all variables, then E1 has a left-absorptive flavor.
Do not stop there. Try to derive E4, E378, or another named collapse law explicitly.

PATTERN 3 — Explicit right-absorptive schema:
If you can derive x = F(x,y,z,...) * x, similarly try to derive E5 or another explicit collapse law.

PATTERN 4 — Finite surjective/injective trick:
In finite magmas, a surjective self-map is injective and vice versa.
If E1 is of the form x = f(y) * g(x,y), then left multiplication by f(y) is surjective in the finite setting.
This can justify cancellation, but only if you name the map and apply the cancellation correctly.

PATTERN 5 — Direct substitution chain:
Try substitutions like:
- y := x
- z := x
- w := x
- y := x*x
- x := y*x
Many true implications come from collapsing E1 to a smaller law by these substitutions.

=== COUNTEREXAMPLE COOKBOOK ===

Use this order:
1. LP via leftmost-leaf comparison.
2. RP via rightmost-leaf comparison.
3. C0 if one side collapses to a variable and the other to 0.
4. XOR / Z3A for affine or parity-sensitive laws.
5. AND / OR / NAND / XNOR for tiny Boolean-style witnesses.

Typical FALSE route:
- E1 passes LP or RP by a leaf check.
- E2 fails the same witness by a different leaf / value check.
- That already settles FALSE. Do not keep searching for a proof.

Typical affine FALSE route:
- Both laws are built from repeated nested binary terms with no absorption pattern.
- XOR or Z3A often refutes by parity / linear mismatch.
- But you must compute, not guess.

=== WHAT HARD2 TAUGHT US TO AVOID ===

DO NOT do any of the following:
1. "LP, RP, XOR, Z3A, C0 all fail E1, therefore TRUE."
   Wrong. That is only weak evidence that E1 is nontrivial.

2. "E2 is singleton-like, so TRUE."
   Wrong unless you proved E1 also implies singleton.

3. "Only XOR satisfies E1 among size-2 magmas."
   Wrong unless you actually checked all 16 size-2 tables.

4. "E1 probably forces a trivial / Boolean / projection structure."
   Wrong unless you derived the actual law that gives that collapse.

5. Writing proof-shaped prose with no equation chain.
   If there is no concrete substitution or rewriting, PROOF must stay empty.

6. Declaring a row/column argument proves triviality after finding only one constant row/column.
   To prove triviality that way, you must show there are enough distinct constant rows/columns to force the whole table to collapse.

=== MINI DERIVATION TEMPLATES ===

Template A — singleton from missing x:
If E1 is x = F(y,z,w) with no x in F, then by transitivity E1 implies x = y.
Once you have singleton, every E2 is TRUE.

Template B — LP counterexample:
Compute leftmost leaves of both sides.
If E1's two leaves match, LP satisfies E1.
If E2's two leaves differ, LP refutes E2.
Then FALSE.

Template C — RP counterexample:
Compute rightmost leaves of both sides.
If E1's two leaves match, RP satisfies E1.
If E2's two leaves differ, RP refutes E2.
Then FALSE.

Template D — constant-zero counterexample:
If under C0 one side of E1 becomes x and the other also becomes x or 0 appropriately,
verify all assignments. Then test E2 on an explicit nonzero assignment.

=== DECISION PROCEDURE ===

Given "Does E1 imply E2?"

1. Parse both equations carefully.
   Identify leftmost leaf, rightmost leaf, and variable multiplicities on BOTH sides of BOTH laws.

2. Run the quick syntactic obstruction checks.
   a. If E2 is singleton-like, note that explicitly.
   b. If multiplicity invariant refutes, answer FALSE.
   c. If LP or RP leaf invariant refutes, answer FALSE.

3. Try cheap substitutions on E1.
   Look for explicit derivations of singleton, E4, E5, E378, idempotence, commutativity,
   or a direct rewrite to E2.

4. Try verified witnesses.
   LP, RP, C0, XOR, Z3A, AND, OR, NAND, XNOR.
   For each candidate: verify E1 fully first; only then test E2.

5. Only answer TRUE if you now have a real proof.
   "No counterexample found" is not a proof.

6. Only answer FALSE if you now have a real counterexample or a valid syntactic obstruction.

7. If you are stuck between unsupported TRUE and unsupported FALSE:
   prefer the side backed by an actual invariant / derivation / witness,
   and never invent algebraic structure that you did not prove.
