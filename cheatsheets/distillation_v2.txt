MAGMA IMPLICATION GUIDE
A magma is a set with one binary operation *.
"E1 implies E2" means every magma satisfying E1 also satisfies E2.

IMPORTANT: To disprove "E1 implies E2", you must find a magma where
E1 is TRUE for ALL variable assignments AND E2 is FALSE for SOME.
If E1 itself FAILS in a magma, that magma is NOT a counterexample.

DECISION RULES (apply in order, stop at first match):

RULE 1 — SINGLETON FORCING → answer TRUE:
If E1 has a bare variable on one side that does NOT appear on the
other side, then E1 forces the magma to have only one element.
A singleton satisfies every equation, so E1 implies ANY E2.
Examples of singleton-forcing E1:
  "x = y * z"           → x not on RHS → TRUE
  "x = y * (z * w)"     → x not on RHS → TRUE
  "x = y * (z * (w * u))" → x not on RHS → TRUE
NON-examples (x appears on both sides):
  "x = x * y"           → x IS on RHS → not singleton
  "x = y * (x * z)"     → x IS on RHS → not singleton

RULE 2 — IDENTITY → answer TRUE:
If E2 is "x = x" (or equivalent), then TRUE, since every equation
implies the trivial identity.

RULE 3 — ABSORPTION → answer TRUE:
If E1 has form "x = x * EXPR" where EXPR contains at least one
variable not equal to x, then E1 implies left-absorption (x = x*y).
Why: substitute any value for the free variables in EXPR.
Similarly "x = EXPR * x" implies right-absorption (x = y*x).
Left-absorption (x = x*y) implies x = x*f for ANY expression f.
So if E1 is "x = x * A" and E2 is "x = x * B", then TRUE.
Dually if E1 is "x = A * x" and E2 is "x = B * x", then TRUE.

RULE 4 — STRUCTURE MISMATCH → answer FALSE:
(a) If E1 is "balanced" (same number of * on each side) but E2 is
    "unbalanced", and E1 is NOT singleton-forcing → FALSE.
(b) If one side of E1 is a bare variable (like "x = ...") but
    both sides of E2 have operations (like "a*b = c*d"), check
    carefully — this is complex and needs magma testing.

RULE 5 — TEST COUNTEREXAMPLE MAGMAS → if any fails, answer FALSE:
A COUNTEREXAMPLE is a magma satisfying E1 but violating E2.
CRITICAL: First check that E1 holds for ALL inputs in the magma.
If E1 fails, that magma tells you NOTHING — try another.

Magma L (left projection, {0,1}): a*b = a.
  Satisfies any equation where leftmost variable matches on both sides.
  Examples: x = x*(y*z), x*y = x*(z*w) are satisfied.
  x = y*x, x*y = y*x are NOT satisfied.

Magma R (right projection, {0,1}): a*b = b.
  Dual of L. Satisfies equations where rightmost variable matches.

Magma C (constant-0, {0,1}): a*b = 0 always.
  Satisfies any equation where both sides have at least one *.
  Does NOT satisfy x = x*y (gives x = 0, false for x=1).

RULE 6 — IF UNCERTAIN, consider these facts:
• About 37% of all implications are TRUE, 63% are FALSE.
• Many equations with bare x on one side and a complex expression
  involving variables not on the other side force singleton → TRUE.
• If E1 has MORE free variables than E2, implication is MORE likely.
  An equation with many free variables is very constraining.
• Equations equivalent to x=y (singleton law) include ALL equations
  where a bare variable appears on one side but not the other.
  31.9% of all equations of order ≤ 4 are equivalent to x=y.
