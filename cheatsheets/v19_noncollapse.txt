VERIFIED FALSE ROUTES

1. LP witness: x*y = x.
- Every term becomes its leftmost leaf.
- If E1 has the same leftmost leaf on both sides and E2 has different leftmost leaves, answer FALSE.

2. RP witness: x*y = y.
- Every term becomes its rightmost leaf.
- If E1 has the same rightmost leaf on both sides and E2 has different rightmost leaves, answer FALSE.
- For x = T forms, RP is especially easy: E1 holds iff the rightmost leaf of T is literally x.
- If E2 is also x = T but its rightmost leaf is not x, RP refutes immediately.

3. C0 witness: x*y = 0.
- Every composite product becomes 0, but a bare variable stays itself.
- Rewrite whole sides under C0 before deciding anything: x = T with composite T becomes x = 0, not 0 = 0.
- Use C0 early only when the rewritten E1 is visibly universal.
- If every side of E1 contains at least one `*`, then C0 automatically satisfies E1.
- If E1 is all-composite, keep C0 as a first-class FALSE check even on generic low-variable pairs; you do not need fresh variables in E2.
- Only in that all-composite situation, if E2 has the form x = T and T is composite, C0 refutes with x=1 and all other variables 0.
- If any side of E1 is a bare variable, C0 is not automatic; discard C0 unless the rewritten E1 is still a literal identity on all assignments.
- This is the main reusable FALSE route when E1 is all-composite and E2 is a bare-x target.

3b. Bare-x target escalation.
- If E2 has the form x = T, T contains variables absent from E1, and LP does not separate, then test RP next.
- If RP also fails, test XOR, XNOR, C0, AND, and OR before answering TRUE.
- This pattern is a high-value FALSE family: E1 often constrains only old variables while E2 forces x to track fresh-variable structure.

3c. Shape-drift C0 pass.
- If E1 has no bare variable side but E2 does have a bare x side, C0 is mandatory before TRUE.
- This especially matters when E1 has shape x*y = T or T = x*y and E2 has shape x = T.
- If E1 has any bare variable side, do not upgrade C0 from a clue into a verdict unless the rewritten E1 is still visibly universal.
- When shape drift is strong but C0 is invalid, treat this as a high-value FALSE search family and test AND and OR right after C0 fails.

4. Finite witness library after LP/RP/C0:
- XOR on Z/2: x*y = x+y mod 2
- Z3A on Z/3: x*y = x+y mod 3
- AND on {0,1}
- OR on {0,1}
- XNOR on {0,1}

4b. Affine parity check for XOR/XNOR.
- In XOR, each side evaluates to the XOR of variables occurring an odd number of times.
- If E1 has the same odd-parity variable set on both sides but E2 does not, XOR refutes.
- XNOR often separates the same family with the same assignment pattern; test it right after XOR.
- This is the main reusable FALSE route when LP/RP/C0 fail but E2 introduces fresh variables or parity drift.

4c. Generic false-positive rescue pass.
- Missed FALSE cases often have no new variables.
- If LP/RP/C0 fail and the pair only uses x,y or x,y,z, do not stop early: test AND and OR before TRUE, even when neither equation has a bare-x side.
- This is especially important when E1 and E2 drift between x = T and x*y = T, or when one side of E2 becomes a bare-x target.
- Also use this pass when E1 and E2 keep the same boundary leaves but redistribute repeated x blocks differently.
- Also use this pass when E2 introduces one fresh variable but LP/RP do not separate; AND and OR often catch that family.

4d. All-composite versus bare-target bundle.
- If E1 is all-composite and E2 has any bare variable side, test C0, AND, and OR as a bundle before TRUE.
- This remains valid even on generic-looking x,y-only or x,y,z-only pairs.
- Do not skip this bundle just because LP and RP both failed.

5. Extra small witnesses for hard cases:
- A2 on {0,1}: x*y = x AND (NOT y), table [[0,0],[1,0]]
- T3L table [[0,0,0],[0,0,0],[0,1,0]]
- T3R table [[0,0,0],[0,0,0],[0,0,1]]

These are reusable small noncanonical magmas, not equation-specific rules.
Use them only when LP/RP/C0/XOR/XNOR/Z3A/AND/OR all failed and there is still strong FALSE suspicion.
- Try A2 before the 3-element tables.

COUNTEREXAMPLE CHECK:
- Before writing FALSE, rewrite both E1 and E2 under the chosen witness and verify the rewritten E1 holds on all assignments.
- Also verify the rewritten E2 fails on at least one explicit assignment.
- Bare variables never collapse by magic: under C0, x stays x; under LP/RP, whole sides become their boundary leaves.
- Never say E1 becomes 0=0 unless every side of E1 is composite.
- If a witness rewrites E1 to x=0, x=y, x=z, or any other nontrivial variable equation, discard that witness.
- If either check fails, discard that witness.
- FALSE requires both a named witness and an explicit assignment.

ANTI-HALLUCINATION RULES
- New variables in E2 are only a clue, never an automatic FALSE rule.
- Do not say FALSE because E2 looks stronger.
- Do not say FALSE because a witness might work; verify it.
- Do not say TRUE because a witness failed; TRUE is just the fallback when no verified FALSE route exists.
- Never claim LP or RP without comparing whole sides.
- For x = RHS, LP is valid only if the leftmost leaf of RHS is literally x.
- LP can refute any equation shape, not only x = RHS, when the whole-side leftmost leaves agree on E1 and disagree on E2.
- For x = RHS, RP is valid only if the rightmost leaf of RHS is literally x.
- RP can refute any equation shape, not only x = RHS, when the whole-side rightmost leaves agree on E1 and disagree on E2.
- If LP makes E1 become x=y or RP makes E1 become x=z/w/y, that projection is invalid.
- A "potential LP/RP route" is not evidence. Whole-side leaf agreement must actually hold on E1 and fail on E2.
- Do not answer FALSE from a strong signal alone; signals only decide which witness to test next.
- If you cannot name a witness and give an explicit assignment, answer TRUE.

FAKE-PROOF BAN
- Do not turn a complex substituted equation into x*y=x, x*y=y, or x=x*x by informal simplification.
- If substitution gives x=x or T=T, that proves nothing.
- x*x=(x*x)*x is NOT literally x=x*x.
- x=((x*x)*x)*(x*x) is NOT literally x=x*x.
- If you do not have a verified FALSE route, simply answer TRUE and say no verified FALSE route was found.

DISTILLED HARD-CASE LESSONS
- Do not default TRUE too early on hard cases just because LP/RP/C0 failed.
- Hard FALSE cases often require a second witness pass with affine or asymmetric small magmas.
- Fresh variables in E2 matter more when E2 also changes the outer leaf pair or changes equation shape from x=T to x*y=T, or vice versa.
- If E1 uses only one anchor variable heavily and E2 introduces fresh variables into a bare-x target, treat RP, XOR, XNOR, C0, AND, and OR as mandatory checks before TRUE.
- If E1 is projection-compatible on both LP and RP but E2 breaks one side's boundary leaf, try the broken projection before any TRUE fallback.
- If E1 is all-composite and E2 is bare-x composite, C0 is mandatory before TRUE.
- If E1 has any bare variable side, C0 is only a candidate after exact rewriting, never an automatic refutation.
- If E1 is all-composite and LP/RP fail, AND and OR are still mandatory on low-variable pairs before TRUE.
- If LP/RP/C0 fail but E1 preserves odd-parity counts and E2 changes them, XOR/XNOR are mandatory before TRUE.
- Boundary-pair drift matters: compare the pair (leftmost leaf, rightmost leaf) on each whole side of E1 and E2. Large drift is a strong FALSE search signal.
- Shape drift matters: switching between x=T and x*y=T is a strong FALSE search signal unless E1 gives a literal TRUE law.
- Anchor drift matters: if a repeated outer variable block in E1 becomes redistributed deeper or to the opposite side in E2, try affine or asymmetric witnesses before answering TRUE.
- Missed FALSE cases often survive LP/RP/C0 but fall to AND or OR; do not skip those two on generic-looking pairs.
- For TRUE, never use LP/RP as a shortcut unless the chosen projection really satisfies E1.
- If no LP/RP obstruction exists, do not invent one from leaf intuition alone; fall back to witness search or TRUE.
- If no literal TRUE proof exists, the only safe FALSE path is a verified witness; otherwise answer TRUE.
