MAGMA IMPLICATION GUIDE v6
E1, E2 are equations over a magma with one binary operation *.
Question: does every magma satisfying E1 also satisfy E2?

CALIBRATION
- Eval is balanced 50/50. Never default to FALSE.
- Say TRUE only from a named proof family below.
- Say FALSE only from a named witness model or safe-false filter below.
- If no TRUE family fired and no witness refuted, that means CONTINUE/UNKNOWN, not FALSE.
- Never invent a custom finite magma. Use only the four witness models listed here.
- A witness model is valid only if E1 holds in that model for ALL assignments.
- One assignment is never enough.

DECISION ORDER
1. SINGLETON SCAN. If it fires, answer TRUE immediately and stop.
2. SAME LAW.
3. TRIVIAL TARGET.
4. LEFT-PROJECTION or RIGHT-PROJECTION.
5. CONSTANT-PRODUCT.
6. SQUARE-BRIDGE.
7. Only now test the four witness models for FALSE.
8. Only after that use safe-false filters.

1. SINGLETON SCAN (MANDATORY FIRST CHECK)
Singleton applies only when the LHS of E1 is a single variable x.
Ask exactly this:
- Is the LHS of E1 a bare variable x?
- Does x appear anywhere in the RHS of E1?

If LHS is a bare variable and x does NOT appear anywhere in the RHS, then E1 is singleton.
Singleton means every E2 is implied. Answer TRUE immediately. Stop.

When singleton fires:
- Do NOT inspect E2.
- Do NOT test witness models.
- Do NOT call it projection.

Examples of singleton E1:
  x = y * z
  x = (y * z) * w
  x = ((y * y) * z) * w
  x = y * ((z * w) * (z * w))
  x = (((y * y) * y) * y) * y
  x = ((y * (z * w)) * u) * v
  x = (y * (z * w)) * (u * v)

Non-singleton examples:
  x = y * (x * z)
  x = x * (y * z)
  x = ((y * x) * z) * w

Important trap:
  x = y * (z * (w * (z * z))) is singleton, not left-projection.
  Projection needs the RHS to literally start with x * or literally end with * x.

2. SAME LAW -> TRUE
If E1 and E2 are the same equation up to consistent variable renaming, TRUE.

3. TRIVIAL TARGET -> TRUE
If E2 is x = x, TRUE.

4. LEFT-PROJECTION FAMILY
Literal gate:
- E1 must have the exact outer form x = x * EXPR.
- If the RHS does not literally begin with x *, left-projection does not apply.

Safe generic lane:
- x does not appear anywhere in EXPR.
- The leftmost variable of EXPR is not x.
- That leftmost variable appears exactly once in EXPR.

Then E1 forces a * b = a.

This generic lane includes:
  x = x * y
  x = x * (y * z)
  x = x * ((y * z) * w)
  x = x * (y * (z * y))
  x = x * (y * ((z * w) * u))
  x = x * ((y * z) * (w * u))
  x = x * ((y * (z * w)) * w)

Extra exact x-in-EXPR lane (allowed only for these exact source shapes and renamings):
  x = x * (y * (x * z))
  x = x * (y * (z * (x * y)))

Consequence under left projection:
- E1 implies E2 iff the leftmost variable on the two sides of E2 is the same.
- Same leftmost variable -> TRUE.
- Different leftmost variable -> FALSE via left projection.

Hard TRUE anchors that are all left-projection:
  x = x * (y * (z * y))       => x = (x * (x * (x * y))) * x
  x = x * ((y * z) * (w * u)) => x = (x * (x * (y * x))) * x
  x = x * ((y * (z * w)) * w) => x * y = (x * (z * w)) * u
  x = x * (y * ((z * w) * u)) => (x * x) * y = (x * y) * z
  x = x * (y * (x * z))       => x = ((x * y) * (z * w)) * u
  x = x * (y * (z * (x * y))) => x = x * (((y * x) * z) * w)

5. RIGHT-PROJECTION FAMILY
Literal gate:
- E1 must have the exact outer form x = EXPR * x.
- The final variable on the RHS must literally be x.
- If the RHS does not literally end with * x, right-projection does not apply.

Safe generic lane:
- x does not appear anywhere in EXPR.
- The leftmost variable of EXPR is not x.
- That leftmost variable appears exactly once in EXPR.

Then E1 forces a * b = b.

This generic lane includes:
  x = y * x
  x = (y * z) * x
  x = ((y * z) * z) * x
  x = ((y * y) * z) * x
  x = ((y * z) * w) * x
  x = (y * (y * (y * z))) * x

Consequence under right projection:
- E1 implies E2 iff the rightmost variable on the two sides of E2 is the same.
- Same rightmost variable -> TRUE.
- Different rightmost variable -> FALSE via right projection.

Hard TRUE anchors that are all right-projection:
  x = ((y * y) * z) * x => x = y * (y * (z * (w * x)))
  x = ((y * z) * w) * x => x = y * ((z * (w * u)) * x)
  x = ((y * z) * z) * x => x = x * ((y * (y * x)) * x)

6. CONSTANT-PRODUCT FAMILY
Use only these exact source shapes and renamings:
  x * y = z * w
  x * y = y * z
  x * y = z * y
  x * x = y * z
  x * y = (y * x) * z
  x * y = (y * z) * x

These force all products to one constant c.
Consequences:
- If both sides of E2 contain *, TRUE.
- If one side of E2 is bare and the other side contains *, FALSE via constant.
- If E2 is x = x, TRUE.

Do not generalize this family to arbitrary product = product equations.

7. SQUARE-BRIDGE FAMILY
Safe source shapes:
  x = y * (x * x)
  x = (y * x) * (y * x)
  x = (y * (y * z)) * (w * x)
  x = (y * (y * x)) * (z * x)
  x = (x * x) * y
  x = (x * y) * (z * z)

Safe TRUE target shapes:
  x = ((x * y) * x) * (z * x)
  x = y * (z * (w * (u * x)))
  x * y = z * (y * (w * y))
  x * x = x * ((y * z) * w)

Apply only when E1 exactly matches a listed source and E2 exactly matches a listed target,
up to variable renaming. Do not generalize.

8. FALSE VIA FOUR WITNESS MODELS ONLY
Use only these four models:

LEFT PROJECTION: a * b = a
- An equation holds iff the leftmost variables on its two sides match.

RIGHT PROJECTION: a * b = b
- An equation holds iff the rightmost variables on its two sides match.

CONSTANT: a * b = 0
- An equation holds iff both sides contain *, or both sides are the same bare variable.

XOR ON {0,1}: a * b = a + b mod 2
- An equation holds iff each variable has the same parity on both sides.

Witness procedure:
- First check whether E1 holds in the witness model using the criterion above.
- If E1 fails in that witness, discard it.
- If E1 holds and E2 fails, answer FALSE and name that witness.
- If no witness refutes, keep going. That is not evidence for FALSE.

Common witness mistakes to avoid:
- Do not build your own operation table.
- Do not test only one assignment.
- Do not say "consider x=0, y=1" as a refutation.
- Do not use the constant witness on singleton E1 unless the constant criterion says E1 holds.
- Do not use left/right projection witnesses unless their hold-criteria actually make E1 true.

9. SAFE FALSE FILTERS (ONLY AFTER 1-8 FAIL)
- If E2 is x = y and E1 is not singleton-forcing, FALSE.
- If E1 is balanced and E2 is unbalanced, and E1 is not in singleton/projection/
  constant/square-bridge families, FALSE.

10. FINAL WARNINGS
- Singleton overrides everything else.
- If singleton fired, answer TRUE immediately.
- Projection families require literal outer shape matching.
- For FALSE, name the witness model or safe-false filter.
- For TRUE, name the proof family.
